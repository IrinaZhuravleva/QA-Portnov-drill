Что такое Тестирование?
Предоставление актуальной информации на соответствие продукта техническому заданию.
Каковы цели тестирования?
Тестирование позволяет оценить качество разрабатываемого продукта.
Что такое QA для ПО?
QA для ПО - это процесс отслеживания и улучшения всех действий, связанных с разработкой ПО, начиная с требований, затем дизайн и код-ревью, заканчивая тестированием и внедрением.
Как расшифровывается QA и SQA?
QA — обеспечение качества; SQA (software quality assurance) - тестирование программного обеспечения
В чем разница между QA и тестировщиком?
Тестировщик работает с продуктом как с результатом (т. е. он предполагает, что именно эта версия ПО попадёт в руки конечного пользователя). QA-инженер работает с продуктом, который находится в процессе создания
Что такое нормы качества?
Это показатели верификации и валидации.
Что такое жизненный цикл разработки ПО(SDLC)?
Концептуальная модель, использованная в проджект-менеджменте, которая описывает стадии, вовлеченные в проект разработки информационной системы, начиная с изучения выполнимости до готового приложения
Основные стадии жизненного цикла разработки ПО?
Анализ требований;
Дизайн (Архитектура);
Разработка (Написание кода);
Тестирование и дебаггинг;
Экcплуатация и Поддержка.
Что такое Agile-разработка ПО?
Гибкая разработка ПО путём сведения разработки к итерациям, каждая - программный проект в миниатюре и включает все задачи, необходимые для выдачи мини-прироста по функциональности.
Что такое Водопадная модель?
Это последовательный процесс разработки ПО, в котором прогресс видится устойчиво текущим стадия за стадией.
Что такое Сборка?
Это скомпилированная версия продукта ПО с самыми актуальными изменениями в коде.
Что такое Версия?
Это уникальный ID, приписываемый продукту ПО для информирования Пользователей о состоянии приложения, его характеристиками.
Что такое Релиз?
Сборка ПО, предоставляемая в конечное пользование.
Как тестировать при отсутствии требований?
Использовать документацию похожих приложений или приложений с похожими характеристиками.
Что такое Валидация?
Проверяет, соответствует ли ПО требованиям и ожиданиям клиента (Заказчика).
Что такое Верификация?
Проверяет, соответствует ли ПО ожиданиям пользователя.
Отличие между Валидацией и Верификацией?
- Валидация отвечает на вопрос “Делаем ли мы правильный продукт?" Верификация отвечает на вопрос “Делаем ли мы продукт правильно?”
Почему требуется тестирование ПО?
Любой сбой, дефект или ошибка, обнаруженные клиентом в готовом продукте, нарушают доверие к компании.
Что такое аудит качества?
Процесс систематической и независимой проверки программного продукта или процесса для оценки соответствия спецификациям, стандартам, соглашениям и другим соответствующим критериям.
Что такое превентивный подход в тестировании?
Этот подход заключается в предотвращении дефектов. При таком подходе тесты разрабатываются на ранних этапах SDLC, то есть до того, как ПО было создано. Он подпадает под анализ качества (QA).
Что такое реактивный подход в тестировании?
Также известен как Validation Process. Заключается в выявлении дефектов. Тесты предназначены для выполнения после того, как ПО было произведено. Подпадает под контроль качества (QC).
Что подразумевается под тестовой средой?
Среда разработки, Среда тестирования, Интеграционная среда, Превью среда, Продакшн среда
Что такое Среда разработки (Development Env)
В ней разработчики пишут код, проводят отладку, исправляют ошибки, выполняют Unit-тестирование. За эту среду отвечают также разработчики
Что такое Среда тестирования (Test Env)
В ней работают тестировщики. Тут тестируются новые билды. Здесь тестировщики проверяют функционал, проводят регрессионные проверки, воспроизводят ошибки.
Что такое Интеграционная среда (Integration Env)
Иногда реализована в рамках среды тестирования, а иногда в рамках превью среды. В этой среде собрана необходимая для end-to-end тестирования схема взаимодействующих друг с другом модулей, систем, продуктов.
Что такое Превью среда (Preview, Preprod Env)
Используется для финальной проверки ПО в условиях максимально приближенным к «боевым».
Что такое Продакшн среда (Prodaction Env)
Среда, в которой работают пользователи.
Что подразумевается под тестовыми данными?
Набор входных значений, необходимых для выполнения Test case. Тестеры определяют данные в соответствии с требованиями.
Основные инструменты тестировщика?
DevTools, Сниффер (Charles Proxy), Тестирование API (Postman или SoapUI), Тестирование производительности (JMeter), Различные инструменты из категории тестирования безопасности (OWASP); мессенджеры, баг-трекинговые системы, генераторы тестовых данных и т.п.
Что такое Мануальное/Ручное тестирование?
Исполнение человеком процедур тестирования без использования инструментов Автоматизированного тестирования и языков программирования.
Что такое Автоматизиро- ванное тестирование?
Процесс написания и выполнения компьютерной программы для тестирования.
Что такое тестирование «черного ящика»?
Тестирование происходит с позиции пользователя (без доступа к использованному исходному коду).
Что такое gray-box тестирование?
Это использование информации о стуктуре, дизайне и/или среды для расширения или фокусировки «black-box тестирование» и для расширения продуктивности тестирования посредством применения уместных методов и инструментов.
Что такое white-box тестирование?
Осуществляется на уровне исходного кода. Основывается на знании внутренней логики кода приложенения.
Что такое Функциональное тестирование?
Оно проводится с целью проверки того, что функции системы работают в соответствии с их назначением. Как правило, функции описаны в рабочих документах (требования, спецификации), но могут быть и не задокументированы.
Что такое Нефункциональное тестирование?
Оно проводится с целью тестирования атрибутов компонента или системы, которые не связаны с функциональностью: Надежность, Эффективность, Удобство пользования, Поддерживаемость и т.д.
Что такое Позитивное тестирование?
Применяется для проверки того, что ПО корректно обрабатывает ситуации, в которых пользователь действует, как от него ожидается (верификация).
Что такое Негативное тестирование?
Применяется для проверки того, что ПО корректно обрабатывает ситуации, в которых пользователь действует некорректно (неправильные действия, вводы, настройки).
Что такое Граничное тестирование?
Для поля "имя" от 3 до 8 знаков: проверка на нижнюю границу. 3 плюс/минус 1 = 2, 3, 4. Проверка на верхнюю = 7,8,9, добавить к проверкам 5 и 15.
Что такое Исследова- тельское тестирование?
Тестировщик "исследует" приложение при отсутствии требований, без заранее спроектированных тестов (выполняемых в точном соответствии с планом), основываясь на своем опыте и креативности. Тестирование черного ящика. Мануальное тестирование.
Что такое Свободное тестирование (ad-hoc testing)?
Выполняется без подготовки к тестированию продукта, без определения ожидаемых результатов, проектирования тестовых сценариев. Это неформальное, импровизационное тестирование. Оно не требует никакой документации или планирования.
Что такое Модульное тестирование (Юнит-тестирование)?
Тестирование "белого ящика" для ПО компонентов/модулей. Производится разработчиками или тестировщиками по методу "белого ящика".
Что такое Интеграционное тестирование?
Тестирование взаимодействия модулей между собой.
Что такое Системное тестирование?
выполняемое на полной, интегрированной системе, с целью проверки соответствия системы исходным требованиям. Альфа-тестирование и бета-тестирование являются его подкатегориями. Это black box testing.
Что такое End-to-end тестирование?
Тестирование приложения полностью как можно ближе реальной среде, на всех уровнях – начиная с фронтенда и заканчивая бэкэндом, включая интерфейс, базы данных и конечные точки.
Что такое Приемочное пользовательское тестирование (UAT)?
Оно проводится группой (SME - subject matter experts) от лица конечных пользователей системы для принятия решения по внедрению.
Что такое Тестирование сборки (Release/Build Acceptance Testing)?
Осуществляется на последней версии сборки/релиза для проверки того, что ПО достаточно хорошо для приемки в тестирование, Это означает, что QA-команда может двигаться от предыдущего релиза к новому.
Что такое Тестирование установки?
Оно проверяет уровень корректности установки ПО продукта в искусственно созданной среде с целью выявления степени его готовности к эксплуатации.
Что такое Тестирование совместимости?
вид нефункционального тестирования - проверка корректной работы продукта в определенном окружении(1. Аппаратная платформа 2. Операционная система (Windows, MacOS, ...) 3. Браузеры (IE, Firefox, Chrome...) 4. Различное разрешение экрана.
Что такое тестирование на Кроссбраузерную совместимость?
Проводится для проверки, что веб-приложение выглядит, ведет себя и взаимодействует с пользователем одинаково в различных браузерах.
Что такое Дымовое тестирование?
рассматривается в качестве короткого цикла тестов, выполняемы для подтверждения факта запуска и выполнения функций устанавливаемого приложения после сборки нового или редактируемого кода. Ориентировано на тестирование большого количества функционала за самые короткие сроки.
Что такое Санитарное тестирование?
Узконаправлено для доказательства того, что конкретная функция работает согласно заявленным в спецификации требованиям. Используется для определения работоспособности определенной части приложения после изменений. Обычно выполняется вручную.
Что такое Регрессионное тестирование?
Собирательное название для всех видов тестирования программного обеспечения, направленных на обнаружение ошибок в уже протестированных участках исходного кода.
Что такое Альфа-тестирование?
Один из типов приемочного тестирования, когда продукт готов на 70% - 90%. Выполняется для имитации поведения в режиме реального времени в соответствии с использованием продукта конечными пользователями на рынкекогда продукт готов на 70% - 90%.
Что такое Бета-тестирование?
Предполагаемый реальный пользователь проверяет продукт на функциональность, удобство использования, надежность и совместимость. Поскольку бета-тестирование происходит на стороне конечного пользователя, это не может быть контролируемым действием.
Что такое Тестирование производитель-ности?
Проводится с целью определения, как быстро работает ПО или его часть под определённой нагрузкой. Также может служить для проверки и подтверждения других атрибутов качества системы, таких как масштабируемость, надёжность и потребление ресурсов.
Что такое Нагрузочное тестирование?
Исследование способности приложения сохранять заданные показатели качества при нагрузке в допустимых пределах и некотором превышении этих пределов (определение «запаса прочности»).
Что такое Стресс-тестирование?
Используется для определения устойчивости системы или модуля в условиях превышения пределов нормальной нагрузки.
Что такое Тестирование на отказ и восстановление?
Проверка на способность противостоять и успешно восстанавливаться после сбоев в связи с ошибками ПО, отказами оборудования или проблемами связи. Проверка систем восстановления (или дублирующих систем), которые при необходимости обеспечат сохранность и целостность данных ПО.
Что такое Юзабилити-тестирование?
Позволяет проверить комфортное использование сайта/ПО для пользователя, насколько легко найти необходимую информацию или выполнить желаемые действия.
Что такое Тестирование доступности?
Один из видов тестирования юзабилити. Проверка, что продукт удобен в использовании, в том числе для людей с инвалидностью или особенностей восприятия(проблемы со зрением, слухом или ограничения в подвижности рук).
Что такое Тестирование безопасности?
Проверка программного продукта на наличие уязвимостей, которыми могут воспользоваться злоумышленники, чтобы получить доступ к закрытой информации или навредить продукту и его пользователям.
Что такое Тестирование локализации?
Тестирование локализации проверяет, должным ли образом программное обеспечение переведено и адаптировано к местным языкам и культурам.
Какой вид тестирования наиболее часто исполняем?
Регрессионное тестирование
Какие бывают техники тест-дизайна?
Эквивалентное Разделение (EP); Анализ Граничных Значений (BVA); Причина / Следствие (CE); Предугадывание ошибки (EG); Исчерпывающее тестирование (ET).
Тест-дизайн черного ящика
– классы эквивалентности; – анализ граничных значений; – таблицы решений; – диаграммы изменения состояния; – тестирование всех пар.
Тест-дизайн, основанный на технике "белого ящика".
Процедура написания или выбора тест-кейсов на основе анализа внутреннего устройства системы или компонента.
Что такое Динамическое тестирование?
Предполагает запуск программного кода. Т. обр., анализируется поведение программы во время ее работы.
Что такое Статическое тестирование?
Предполагает, что программный код во время тестирования не будет выполняться. При этом само тестирование может быть как ручным, так и автоматизированным.
Что такое Тестирование взаимодействия?
Нацелен на оценку качества взаимодействия компонент программной системы или всего приложения с другими компонентами или программным обеспечением.
Что такое баг?
Несоответствие между актуальным поведением ПО приложенения и его ожидаемым поведением. Мы узнаем об ожидаемом поведении из требованийБ спецификаций и др. технической документации.
Что такое правило трех W?
Что (What) случилось? Где (where) случилось? При каких (which) обстоятельствах?
Наиболее важные компоненты в Bug Report?
Короткое описание (Заголовок);
Подробное описание;
Шаги для воспроизведения;
Результат и Ожидаемый результат;
Вложения;
Приоритет дефекта;
Серьезность дефекта;
Статус;
Окружение дефекта.
Что такое Жизненный цикл бага?
Новый (New);
Открыт (Opened);
Отложен (Deferred);
Отклонен (Rejected);
Дубликат (Duplicate) уже был ранее внесен в «Bug-tracking»;
Назначен (Assigned);
Исправлен (Fixed);
Проверен (Verified);
Повторно открыт (Reopened).;
Закрыт (Closed)
Когда надо писать баг-репорт?
Сразу же при обнаружении бага.
Почему надо писать баг-репорты?
Для исправления багов.
Что делает баг-репорт хорошим?
Следование правилу Трех W;
Описание, как воспроизвести баг;
Анализ проблему для минимизации шагов воспроизведения бага;
Полный, легко понятный, непротиворечивый.
Кто может изменить Назначение/Приоритет в баг-репорте?
Тестировщик назнает серьезность;
Development Manager назнает Приоритет.
Назовите уровни Важности (Severity) проблемы?
S1 Блокирующая (Blocker);
 S2 Критическая (Critical);
S3 Значительная (Major);
S4 Незначительная (Minor);
S5 Неудобство (Tweak);
S6 Текст/опечатка (Text);
S7 Тривиальная (Trivial);
Назовите уровни Приоритета бага.
P1 Высокий (High);
P2 Средний (Medium);
P3 Низкий (Low).
Какой тип тестирования результатов находит наибольшее количество багов?
Негативное тестирование (vs Позитивное тестирование того же типа).
Назовите баг-трекеры?
Jira;
Bugzilla;
Mantis.
Статусы багов?
Open;
Pending;
Fixed/Resolved;
Closed;
Can't Reproduce;
Deferred;
Если разработчик не может воспроизвести баг?
Воспроизведите на своем компьютере; затем еще на 1-3 компьютерах; если сможете: сравните свою рабочую среду со средой разработчика; попросите разработчика продебажить в среде QA с помощью его/её инструментов.
Если разработчик пометил ваш баг, как "Как в дизайне"?
Проверьте требования; Если не существует специфических требований, сравните ту же особенность(feature), внедренную в приложения качества или поговорите со своим менеджером и/или разработчиком; Закройте баг.
Для чего созданы баг-трекинговые системы?
Сообщать о багах;
Отслеживать (изменять) баги;
Закрывать баги;
Проводить статистический анализ, отслеживать и суммировать результаты;
Работать над улучшениями качества.
Зачем нужна База данных для трекинга багов?
Учет;
Средство коммуникация (между сотрудниками и между отделами);
Организация информации;
Отслеживание персонального выполнения.
Первоочередная задача Базы данных для трекинга багов?
Исправление багов.
Назовите Принципы тестирования
1. тестирование показывает наличие дефектов; 2. исчерпывающее тестирование невозможно; 3. раннее тестирование; 4. скопление дефектов; 5. парадокс пестицида; 6. тестирование зависит от контекста; 7. заблуждение об отсутствии ошибок.
.
Что такое Тест-план?
Документ, описывающий весь объем работ по тестированию, начиная с описания объекта тестирования, стратегии, расписания, критериев начала и окончания тестирования, до необходимого в процессе работы оборудования, специальных знаний, а также оценки рисков с вариантами их разрешения.
Что такое Тест-кейс?
Описание проверки работы системы, которое может выполнить любой человек из команды, будь то тестировщик, разработчик, аналитик или даже бизнес-заказчик.
Что такое Use Case?
Описание требования пользователя к решению определенной задачи в системе, описания взаимодействия пользователя с системой. Позволяет обеспечить полноту функциональных требований, следующих из потребностей пользователей.
Наиболее важные компоненты тест-кейсов?
Уникальный идентификатор (ID);
Название — краткое описание сути проверки;
Предварительные шаги (если есть);
Шаги;
Ожидаемый результат (ОР).
Зачем мы используем тест-кейсы?
Для лучшего тестового покрытия;
Могут быть использованы повторно;
могут быть пересмотрены;
Отслеживаемы - знаем, что тестируем;
Трекинг результатов тестов;
Creates a deliverable.
Что такое Тестовая матрица?
Механизм сбора данных. Представляет структуру для тестирования эффекта сочетаемости двух или более переменных, стечения обстоятельств, типов технического обеспечения или событий
Что такое Тестовый набор (Test Suite)?
Совокупность тест-кейсов, выбранных с некоторой общей целью или по некоторому общему признаку (нпрм, проверка ЛК.). Иногда в такой совокупности результаты завершения одного тест-кейса становятся входным состоянием приложения для следую- щего тест-кейса.
Что такое Итоговый отчет о тестировании (test summary report)?
Подводит итог задачам и результатам тестирования и содержит оценку соответствующих объектов тестирования относительно критериев выхода. -какие задачи были протестированы (функциональности); -сроки проведения тестирования, кем проводилось тестирование; -найденные ошибки (с описанием критических ошибок) -исправлены ли эти ошибки на дату подачи отчета; -обратить внимание на часто повторяющиеся ошибки (если такие есть); -по необходимости добавить приложение со всеми тест-кейсами
Что такое Матрица соответствия требований (Traceability Matrix)?
Таблица с соответствием функциональных требований (functional requirements) ПО и подготовленных тестовых сценариев (test cases). В заголовках колонок таблицы - требования, а в заголовках строк — тестовые сценарии. На пересечении — отметка, означающая, что требование текущей колонки покрыто тестовым сценарием текущей строки.
Что такое Анализ рисков (Risk Analysis)?
Подразумевает действия, предпринимаемы для избежания неправильных ситуаций на этапе разработки проекта ПО, которые могли бы негативно повлиять на цель, качество, график или стоимость проекта.
Когда понятно, что сделано достаточно тестов?
Когда процесс тестирования приходит в точку, где дополнительные тесты не будут значительно влиять на качество ПО.
К каким документам следует обращаться при создании Тест-кейсов?
Ко всей возможно доступной деловой и технической документации: PRD - Product Requirements Document; Бизнес-требования (Business Requirements Document); Functional Specifications; Manuals and Help; Use Cases; Test Design.
Что такое Бизнес-требования (BRD)?
Этот документ пишется бизнес-аналитиками и детализирует бизнес-решения для проекта, включая документацию с запросами и ожиданиями клиента.
В чем разница между тест-кейсом и тест-планом?
Тест-кейс - самая маленькая единица, имеющая дело с одним требованием. Тест-план - наоборот, содержит цели, подход и фокус: какие компоненты ПО будут тестироваться, какие виды тестов будут проводиться и что должно быть для этого сделано, порядок выполнения тестов.
Что такое Продуктная документация (Product/ Development Documentation)?
Плана проекта (Project Management Plan), в т. ч. и тест-план; Требования к программному продукту (Product requirements Document (PRD)) и функциональные спецификации; Архитектуру и Дизайн; Тест-кейсы и наборы тест-кейсов (tesr cases, test suites); Технические спецификации (Technical Specification)
Что такое Проектная документация (Project Documentation)?
Включает в себя продуктную документацию; Пользовательскую и сопроводительную документацию; Маркетинговую документацию.
Что такое чек-лист?
Список, содержащий ряд необходимых проверок во время тестирования ПО. Отмечая пункты списка, команда или один тестировщик могут узнать о текущем состоянии выполненной работы и о качестве ПО.
Что такое HTML?
HTML - это язык гипертекстовой разметки для создания веб-страниц.
Что такое HTML-теги?
Это ключевые слова, обычно парные и окруженные скобками. Первый тег - открывающий, второй - закрывающий со слэшем.
Что такое JavaScript?
Это язык программирования, на котором пишется код для придания интерактивности веб-страницам.
Java и JavaScript одно и то же?
Полностью разные языки по концепту и дизайну. Java - более мощный и сложный язык программирования.

В чем разница между == и === в JavaScript?
Оператор == сравнивает на равенство, а === — на идентичность. Плюс оператора === состоит в том, что он не приводит два значения к одному типу. .
Что такое SQL?
«Язык структурированных запросов» применяется для создания, модификации и управления данными в реляционной базе данных, управляемой соответствующей системой управления БД.
Что такое База данных?
Это хранилище для контента/информации (данных), систематизированных таким образом, чтобы эти материалы могли быть найдены и обработаны.
Что такое Query?
Функция QUERY позволяет сделать выборку нужных строк из таблицы с помощью SQL-запроса и отсортировать их.
Что такое DBMS?
Система управления базами данных "СУБД" предназначена для определения, создания, выборки, обновления и администрирования баз данных.
Что такое Primary Key?
Это уникальный идентификатор для каждой записи в таблице БД.
Что такое Внешний ключ (Foreign Key)?
Это колонка (или комбинация колонок), которая используется для установления отношения между таблицами.

Что такое XML?
eXtensible Markup Language — расширяемый язык разметки;
Был создан для передачи, а не для показа данных

Что такое CSS?
«Каскадные таблицы стилей» — формальный язык описания внешнего вида документа (веб-страницы);
Определяет внешний вид HTML-элементов.
Что такое FireBug?
Отладчик web-приложений, используется как отдельное расширение для браузера Mozilla Firefox, являющееся консолью, отладчиком, и DOM-инспектором.
Что такое Cookies?
Небольшой фрагмент данных, отправленный веб-сервером и хранимый на компьютере пользователя: аутентификации, настроек, отслеживания состояния сеанса доступа.
Что такое HTTP?
«Протокол передачи гипертекста» — протокол прикладного уровня передачи данных, изначально — в виде гипертекстовых документов в формате HTML, в настоящее время используется для передачи произвольных данных.
Что такое IP-адрес?
Internet Protocol Address «адрес Интернет-протокола» — уникальный цифровой адрес (label), предписанный каждому устройству (компьютер, принтер), участвующему в компьютерной сети, которая использует для связи интернет-протокол.
Что такое VPN?
«Виртуальная частная сеть» — обобщённое название технологий, обеспечивающих одно или несколько сетевых соединений (логическую сеть) поверх другой сети (нпрм., Интернет). Например, соединить сотрудников с их домашних устройств с сетью их компании через безопасный доступ.
Что такое Selenium Webdriver?
Программная библиотека, которая позволяет разрабатывать программы, управляющие поведением браузера.
Что такое XPath?
Язык запросов к элементам XML-документа. Применяется для навигации между элементами и атрибутами в XML-документе.
Что такое DOM Inspector?
Это инструмент, который можно использовать для инспекции и редактирования DOM или любого веб-документа или XUL-приложения (XML User Interface Language).
Что такое JSON?
JavaScript Object Notation - текстовый формат обмена данными, основанный на JavaScript.
Что такое jQuery?
Мультибраузерная JS библиотека, предназначенная для упрощения написания скриптов на клиентской части HTML.
Что такое PHP?
Язык программирования общего назначения с открытым исходным кодом. PHP специально сконструирован для веб-разработок и его код может внедряться непосредственно в HTML.
Что такое API?
Application Programming Interface — это составляющая часть сервера, которая получает запросы и отправляет ответы. Каждый раз, когда пользователь посещает какую-либо страницу в сети, он взаимодействует с API удалённого сервера.
Что такое REST API?
REST (RESTful) - это общие принципы организации взаимодействия приложения/сайта с сервером посредством протокола HTTP. Особенность REST в том, что сервер не запоминает состояние пользователя между запросами - в каждом запросе передаётся информация, идентифицирующая пользователя (например, token, полученный через OAuth-авторизацию) и все параметры, необходимые для выполнения операции.
Методы HTTP запросов?
Основное предназначение запросов:
1. получение - GET
2. добавление - POST
3. модификация - PUT
4. удаление - DELETE PUT-запрос /rest/users/125 - изменение информации о пользователе с id=125
??Взаимодействие с сервером REST (RESTful)?
взаимодействие с сервером сводится к операциям 1. получение данных с сервера (обычно в формате JSON, или XML) 2. добавление новых данных на сервер 3. модификация существующих данных на сервере 4. удаление данных на сервере
Чем метод POST отличается от PUT?
POST подразумевает, что вы передаёте данные в теле запроса. PUT - вы передаёте всё, что нужно в URL. Тела запроса нет.
Чем отличается HTTP от HTTPS?
передаваемые по протоколу HTTP данные не защищены, HTTPS обеспечивает конфиденциальность информации путем ее шифрования; HTTP использует порт 80, HTTPS — порт 443.
